# Site Notes Overlay

A Chrome extension that gives every website its own sticky notes. Notes appear as
a translucent, draggable overlay whenever you visit a site you've written notes
for.

## Features

- **Multiple notes per site** — a scrollable stack of cards, each with its own title
- **Site or page scope** — keep a note across a whole site, or pin it to one URL
- **Color coding** — six colors, with a per-site default
- **Cross-device sync** — notes ride along via `chrome.storage.sync`; oversized
  notes stay local-only and are marked "⚠ local"
- **All-notes dashboard** — search every note across every site and jump to any of them
- **Export / import** — back up notes to JSON (per-site or global) and restore them
- **Auto light/dark** — matches your system theme
- Adjustable transparency, goes fully opaque on hover; draggable, resizable, and
  collapsible panel that remembers its position/size per site
- Autosaves as you type; toggle with `Alt+N` or the toolbar icon

## Develop

```bash
npm install
npm run build      # bundles src/ → content.js + background.js
npm run watch      # rebuild on change
npm test           # storage-layer unit tests (no browser needed)
npm run package    # build + zip for the Chrome Web Store
```

## Install (unpacked)

1. `npm install && npm run build`
2. Open `chrome://extensions` and enable **Developer mode**
3. Click **Load unpacked** and select the project folder

After editing source, run `npm run build` and click the extension's ⟳ reload in
`chrome://extensions`. Works on Chrome, Edge, and Brave. Content scripts can't run
on `chrome://` pages or the Chrome Web Store, so the overlay won't appear there.


## How it works

Each note is stored as its own item so it fits `chrome.storage.sync`'s per-item
limit:

```
siteNotes:meta:<host>  → { defaultColor, noteIds }
siteNotes:note:<id>    → { id, host, scope, url, title, text, color, ... }
siteNotes:ui:<host>    → { x, y, w, h, opacity, collapsed, open }
```

Small notes are mirrored to `chrome.storage.sync` (coalesced writes stay under
the rate limit); notes are merged across local and sync with newest-wins. Panel
geometry and per-site defaults stay device-local. A one-time migration upgrades
the v1 single-note key, preserving both text and panel position.
